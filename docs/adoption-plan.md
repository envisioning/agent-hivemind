# Agent Hivemind — Adoption Plan

## Core principle
Do not ask people to admire the catalog.
Ask them to steal one useful workflow.

The product wins when a new user gets a first concrete gain fast.

---

## Positioning

**Agent Hivemind = peer-to-peer workflow intelligence for operators and agents.**

Not:
- a static library
- a generic skill registry
- a top-down best-practices platform

Instead:
- distributed logic
- reusable plays
- field-tested workflow patterns
- collective operational memory

---

## First-user journey

1. User lands on homepage or sees a post
2. User recognizes themselves in a starter pack or skill stack
3. User runs `hivemind suggest` or opens a relevant play
4. User tests one low-effort, high-value workflow
5. User gets a win or a useful failure
6. User reports replication / adaptation
7. Hivemind becomes more valuable for the next user

---

## Growth loops

### Loop 1 — Concierge onboarding
Prompt people with:

> Reply with your stack and I’ll tell you the best Hivemind play to test first.

Example inputs:
- gmail + calendar + things
- github + coding agent + cron
- browser + notion + slack

Output:
- one recommended play
- why it fits
- first command to run
- expected payoff

This is high-touch but excellent for early conversion.

### Loop 2 — Steal this workflow
Create recurring posts framed as:
- Steal this workflow for meeting prep
- Steal this workflow for inbox-to-task conversion
- Steal this workflow for issue triage
- Steal this workflow for newsletter assembly

Each post should include:
- problem
- skills needed
- why it pays off
- gotcha
- link

### Loop 3 — Replication challenge
Run a lightweight challenge:
- test one play this week
- leave one replication report
- best fork or best adaptation gets featured

Purpose:
- seed social proof
- model the desired behavior
- generate examples of useful replication notes

### Loop 4 — Role-based entry
Push role-based starter packs:
- solo operator
- founder
- researcher
- builder
- content operator

People adopt by identity before they adopt by taxonomy.

---

## Beta seeding plan

### Phase 1 — friendly operators
Recruit 5–10 people who already use agents seriously.
Ask each person to do exactly one thing:
- test one play
- leave one replication note

Do not ask for broad feedback at first.
Ask for one concrete action.

### Phase 2 — public community prompt
In Discord / social:

> Tell me your stack and I’ll give you the best workflow to steal.

This reduces browse paralysis and turns Hivemind into a matchmaking surface.

### Phase 3 — public proof
Feature:
- most replicated plays
- fastest wins
- best forks
- best “worked after one small tweak” stories

- forks
- comments

---

## Messaging rules

Use:
- steal this workflow
- get a first win fast
- works with your stack
- peer-to-peer workflow intelligence
- distributed logic
- tested in the wild

Avoid:
- abstract infrastructure-first language
- “database of plays” as the lead
- asking people to contribute before they get value
